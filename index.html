Order Execution Engine

Designed and implemented an order execution backend focused on Market Orders, demonstrating realistic execution flow, routing logic, retries, and real-time lifecycle visibility using a mocked DEX environment.

Orders are submitted via a Fastify REST API and processed asynchronously using a BullMQ-backed worker. The system routes between mocked Raydium and Meteora routers, selects the best net quote (price minus fee), applies optional slippage checks, and simulates execution delays and transaction hashes.

Each order transitions through well-defined lifecycle states (pending, routing, building, submitted, confirmed, failed), which are persisted to PostgreSQL and streamed live to clients over WebSockets.

  • Node.js, Fastify
  • PostgreSQL (order persistence)
  • Redis + BullMQ (queue, retries, concurrency)
  • WebSockets (lifecycle streaming)
  • Mock DEX routers (Raydium, Meteora)
